The Future of Work from First Principles

Projecting 2 / 5 / 10 years out from the LLM and agent axioms - what changes, what stays invariant.

Topic folder: first-principles-tutor/future-of-work · Captured 2026-07-11 · Builds on ../llms/, ../agents/, ../psychology/, ../behavioral-economics/

0TL;DR

  • As generation goes to zero, verification and judgment become the scarce goods. Humans move up the stack from producer to director/verifier.
  • Value migrates to work where generation is cheap but verification is hard.
  • Agents become soft microservices: small, single-responsibility, composable - built from routers + specialists + tools.
  • The durable bets are the 5 invariants: verification/judgment, accountability, taste, trust, physical embodiment.
  • Best move: own the domain + workflow + trust, rent the intelligence, build for falling inference costs, price for outcomes.

1The meta-principle: layers stack

The engine of the whole exercise

The mechanisms that fall out of one layer become the primitives (local axioms) of the next. LLM mechanisms became the axioms of agents; psychology's became behavioral economics'. This is layered abstraction + emergence ("more is different"). Knowledge and technology compound because each generation's emergent mechanisms become the next generation's raw materials. "Cheap generation" is itself a new mechanism that becomes the base axiom of the next economy.

2Verification economics (does the cost go up?)

"Cost" hides two things:

  • Per-item verification cost tends to fall (better tools, tests, agents checking agents).
  • Total verification burden rises - generation volume explodes, so aggregate checking effort goes up even as unit cost drops.

Amdahl's Law applied to work: automate 70% of a task and the remaining 30% (verification + judgment) now dominates the time - and therefore the value. Whatever you do NOT automate becomes the bottleneck and captures the economics.

A P-vs-NP intuition for labor

Value migrates to tasks where generation is cheap but verification is hard.
Cheap to verify (does it compile?) -> fully automatable, low value.
Hard to verify (is this strategy sound?) -> stays human, high value.

The skill ladder of leverage (each rung scarcer)

  1. Do the task (being automated)
  2. Specify + verify the task (the near-term premium)
  3. Decompose - break a fuzzy goal into verifiable, tool-sized pieces (the meta-skill)
  4. Build the agent that does the piece
  5. Optimize agent-building - the systems/evals that build and improve agents (agents building agents)
  6. Set the goals for that system (taste + accountability - the top)
Design corollary

Winners engineer verifiability: structure work so both generation AND checking are cheap - small reviewable diffs, typed/structured outputs, tests, contracts. "Make it easy to verify" is a core design skill. Decomposition and agent-building are the disciplines that convert hard-to-verify work into easy-to-verify pieces.

3Agents as (soft) microservices

Microservice principleAgent equivalent
Single responsibilityOne bounded capability (Caddie = BDM analysis, nothing else)
Well-defined interfaceInput goal -> output result (ideally a schema)
Independently deployablePublish/version one agent (the marketplace)
ComposableOne agent calls another (invoke_agent)
Encapsulated stateIts own context/memory window

It is fractal: a single agent is often internally router -> specialists -> tools (the PRISM pattern). Three differences from classic microservices, each a design constraint:

  • Non-deterministic -> you need evals, not just unit tests.
  • Natural-language interfaces (fuzzy contracts) -> lossy hand-offs; tighten with structured outputs.
  • Expensive per call -> composition has real cost; don't over-chain.
SOLID, applied

S: small sharp agents beat god-agents. I: narrow tool allow-lists (least privilege). D: agents depend on tool contracts, not implementations. DRY: factor shared instructions into skills, don't copy prompts. YAGNI: no "just in case" tools (attack surface + distraction).

4How agents talk, and how to organize them

First principle: every hop is lossy - like telephone, or repeated JPEG compression. Each agent re-interprets the last one's words through its own probabilistic lens. So:

  • Minimize hops - each one compounds error and cost.
  • Tighten contracts at boundaries - pass structured data, not prose, where you can.
  • Return summaries, not raw dumps - compaction at the boundary.
  • Give each sub-agent its own fresh context - sub-agents are a context-isolation strategy; isolation IS memory management.
  • Separate the doer from the critic - a distinct verifier beats self-checking (likely != true).

Coordination patterns

  • Hub-and-spoke (router -> specialists) - best control + observability. Default.
  • Pipeline (A->B->C) - linear workflows.
  • Blackboard (shared memory) - agents read/write common state.
  • Debate/critic - generator + adversarial checker for high-stakes output.

Folder / organization for optimization

  • Organize by capability, not project (single responsibility).
  • Namespaced, flat, discoverable (mp-prism-*, mkplc-prism-*) so a router can enumerate the roster.
  • Separate the layers: routers · specialists · skills (shared knowledge, DRY) · tools.
  • A registry/catalog for discovery (the marketplace).
  • Version + trace everything - optimize by observing runs (evals/logs per agent).
  • Objective = minimize three things: duplication (DRY via skills), context per agent (leaner = faster, cheaper, more accurate), and hops (less telephone).

5The 5 invariants (what stays constant)

An invariant is durable because a deeper layer (law, psychology, physics) protects it from cheap intelligence.

  1. Verification & judgment. Why: the likely-vs-true gap is intrinsic; and judgment (choosing under conflicting values + uncertainty) is not a prediction problem. Premium: being the trusted final check.
  2. Accountability. Why: legal + social systems require someone who can bear consequences. An agent can't be sued, fired, or feel shame - it can't hold responsibility because it can't suffer the downside (moral hazard, structural). Premium: signing your name to outcomes.
  3. Taste / goal-setting. Why: Hume's is/ought gap - you can't derive what ought to be done from data about what is. Premium: deciding what is worth doing and what "good" looks like.
  4. Trust & relationships. Why: psychology axiom #4 - we are obligately social; for high-stakes and care work humans want a human. Premium: relationships, credibility, care, persuasion.
  5. Physical-world embodiment. Why: atoms are harder than bits; dexterity, presence, and real-world liability lag digital cognition. Premium: work rooted in the physical and in-person.
Honesty flag

These are durable relative to the current wave, not eternal. Robotics could erode #5; better tooling narrows #1. Watch them; don't worship them. (See the "limits of first principles" section in the LLM note.)

6The 2 / 5 / 10 year horizons

Grounded in two consequences: LLM axioms -> cost of plausible output -> 0 (but likely != true, so verification is permanent); agent axioms -> cost of multi-step goal-pursuit collapsing (automation moves from tasks to processes).

~2 years (fairly confident)

  • Agents reliable for bounded, well-specified tasks. Your role: specifier + reviewer.
  • Every knowledge worker manages 1-3 agents; prompting/orchestration = basic literacy.
  • Bottleneck moves to verification (generate faster than you can check) and context-plumbing (right data + permissions to the agent).
  • Jobs unbundle: routine ~60-70% automates; judgment ~30% is amplified.

~5 years (moderate uncertainty)

  • Multi-agent systems / "agent orgs." Humans manage fleets - editors, managers, verifiers.
  • Work reorganizes around specify -> delegate -> verify. Scarce skills: decomposition, evaluation, taste.
  • Small teams do what large ones did; org charts flatten. Infrastructure matures around eval, observability, permissions, memory.

~10 years (high uncertainty - speculative)

  • Unit of work may become a goal handed to a swarm. Humans concentrate at the top (what/why/accountability) and edges (physical, high-trust, care).
  • Generic knowledge work commoditizes; premium shifts to the 5 invariants.
  • Genuinely unpredictable: regulation, model ceilings, distribution of gains. Emergence means some of this cannot be derived - it must be watched.

7The investment thesis

Invest in what stays constant: the invariants and the top of the stack; rent what is commoditizing.

  • Start building simple agents now; grow systems you can monetize. Why: the skill compounds (you learn decomposition/verification/orchestration only by doing); and your agents are appreciating assets - they get better for free as the models underneath improve.
  • The moat is around the model, not the model. Own domain knowledge + workflow + proprietary data/feedback loop + trust. Models commoditize; your domain eval-set, relationships, and distribution do not.
  • Thin wrappers die; deep wrappers win. A prompt with no moat gets crushed; a tool that owns a domain's data, workflow, verification, and trust captures the value of cheap intelligence.
  • Price for outcomes + accountability, not tokens. Value = (work displaced or enabled) x (trust that it is correct). Pricing power lives in the verification/trust layer.
One line

Own the domain + workflow + trust, rent the intelligence, build for falling inference costs, and price for outcomes + verification - because those are the parts the first principles say will still matter in ten years. (Compute economics detailed in the "Economics of AI" note.)

8The behavioral overlay (why we built all the notes)

The transition itself is governed by the psychology + behavioral-econ notes:

  • Loss aversion -> people fear what they'll lose more than value what they'll gain; adoption lags rationality.
  • Herding -> orgs wait... then stampede.
  • Present bias -> short-term ROI pressure starves long-horizon bets.
  • Moral hazard -> delegating to agents that act for you is a textbook principal-agent problem -> the future needs approval gates, audit trails, incentive redesign.
Synthesis

LLMs + Agents + AI tell you what becomes possible; Psychology + Behavioral Economics tell you how fast humans will let it happen. The notes are not separate - they are one stacked argument.
