# Engineering Course Explainer (India)

This project turns raw branch names from counselling data into an actual decision-support content product.

Instead of forcing students to guess what branches like **Engineering Physics**, **Materials Science**, or **Electronics and Instrumentation** actually mean, this library explains:

- what you study,
- what kind of work graduates do,
- what adjacent branches are similar,
- and how to think about course fit beyond just cutoff obsession.

## Why this matters

Indian counselling workflows are overloaded with abbreviations, specialization variants, and brand-vs-branch confusion.
Students often optimize for names they recognize, not for work they would actually enjoy.

This explainer helps make branch choice more legible.

## Product lens

The value is not “content for content’s sake.”
The value is reducing ambiguity at a high-stakes decision point.

That means every course page should help answer:

1. What will I actually study?
2. What problems will I learn to solve?
3. What jobs or industries does this realistically connect to?
4. What similar branches should I compare before finalizing preferences?

## Current scope

- Similar/adjacent branch hints
